# ДИПЛОМНАЯ РАБОТА

## ТЕМА ДИПЛОМНОЙ РАБОТЫ

**"Разработка защищённой микросервисной платформы для управления лабораторными работами по информационной безопасности"**

---

## АКТУАЛЬНОСТЬ ТЕМЫ

В условиях стремительной цифровизации всех сфер общественной жизни и возрастания угроз информационной безопасности особую актуальность приобретает подготовка высококвалифицированных специалистов в области кибербезопасности. Согласно данным международных исследований, к 2025 году мировой дефицит специалистов по информационной безопасности составит более 3,5 миллионов человек.

В Российской Федерации подготовка кадров по информационной безопасности приобретает стратегическое значение в контексте реализации национальной программы "Цифровая экономика Российской Федерации", федеральных образовательных стандартов высшего образования и государственной политики в области информационной безопасности.

Современные образовательные учреждения сталкиваются с рядом критических проблем при организации практического обучения информационной безопасности:

- **Недостаток практических навыков**: Студенты получают теоретические знания без возможности работы с реальными уязвимыми системами и инструментами
- **Ограниченность материально-технической базы**: Высокая стоимость специализированного оборудования и программного обеспечения для создания лабораторных стендов
- **Сложность обеспечения изоляции**: Трудности создания безопасных изолированных сред для практических занятий
- **Проблемы масштабируемости**: Невозможность одновременного проведения лабораторных работ для больших групп студентов
- **Быстрое устаревание контента**: Динамично развивающиеся киберугрозы требуют постоянного обновления учебных материалов

Традиционные подходы к организации лабораторных работ по информационной безопасности не соответствуют современным требованиям образовательного процесса и не обеспечивают необходимый уровень практической подготовки студентов.

---

## ПРАКТИЧЕСКАЯ ЗНАЧИМОСТЬ

### Для системы высшего образования Российской Федерации:

1. **Повышение качества подготовки специалистов**: Создание современной цифровой лабораторной базы, обеспечивающей практические навыки работы с реальными системами информационной безопасности

2. **Стандартизация образовательного процесса**: Унификация подходов к практическому обучению информационной безопасности в соответствии с федеральными образовательными стандартами

3. **Обеспечение доступности образования**: Возможность проведения лабораторных работ в дистанционном формате, что особенно актуально в условиях современных вызовов

4. **Соответствие требованиям ФГОС**: Полное соответствие федеральным государственным образовательным стандартам по направлениям подготовки в области информационной безопасности

### Для студентов высших учебных заведений:

1. **Формирование практических компетенций**: Получение опыта работы с реальными уязвимыми системами, инструментами анализа и защиты информации

2. **Повышение конкурентоспособности на рынке труда**: Соответствие требованиям российских IT-компаний и готовность к работе в условиях импортозамещения

3. **Обеспечение гибкости обучения**: Возможность выполнения лабораторных работ в любое время и с любого места при наличии доступа к сети Интернет

4. **Индивидуализация образовательного процесса**: Адаптация лабораторных работ под индивидуальные потребности и темп обучения каждого студента

### Для профессорско-преподавательского состава:

1. **Автоматизация учебного процесса**: Упрощение создания, настройки и управления лабораторными работами

2. **Мониторинг образовательных результатов**: Возможность отслеживания прогресса обучения студентов и анализа эффективности образовательного процесса

3. **Актуализация учебного контента**: Быстрое обновление лабораторных работ в соответствии с новыми угрозами и технологиями информационной безопасности

4. **Методическая поддержка**: Предоставление готовых шаблонов, методических материалов и рекомендаций по проведению занятий

### Для IT-индустрии Российской Федерации:

1. **Подготовка квалифицированных кадров**: Формирование пула специалистов по информационной безопасности, готовых к работе с отечественными технологическими решениями

2. **Поддержка импортозамещения**: Подготовка специалистов для работы с российским программным обеспечением и технологиями

3. **Стимулирование инноваций**: Развитие отечественных технологий в области информационной безопасности и образовательных платформ

4. **Экспорт образовательных услуг**: Возможность использования разработанной платформы для международного сотрудничества и экспорта российского образования

## ОЖИДАЕМЫЕ РЕЗУЛЬТАТЫ

### 1. Теоретические результаты:

- **Методология проектирования микросервисных образовательных платформ**: Разработка принципов и подходов к созданию масштабируемых, безопасных и высокопроизводительных систем для управления образовательными процессами

- **Архитектурные решения для образовательных систем**: Создание типовых архитектурных паттернов, адаптированных для специфики образовательной деятельности

- **Модель ролевого доступа для образовательных платформ**: Разработка гибкой системы управления правами доступа, учитывающей особенности образовательного процесса

- **Алгоритмы мониторинга образовательной деятельности**: Создание специализированных метрик и индикаторов для оценки эффективности образовательного процесса

### 2. Практические результаты:

#### 2.1 Микросервисная платформа:
- **Backend-сервисы на базе FastAPI**: Реализация набора микросервисов для управления лабораторными работами, пользователями, авторизацией и мониторингом
- **API Gateway**: Система маршрутизации и балансировки нагрузки для обеспечения высокой доступности и производительности
- **Сервис авторизации**: Интеграция с Keycloak для централизованной аутентификации и авторизации пользователей
- **Сервис управления данными**: Система кэширования через Redis и асинхронной обработки задач через RabbitMQ и Celery

#### 2.2 Система мониторинга и безопасности:
- **Мониторинг микросервисов**: Интеграция Prometheus и Grafana для сбора метрик, создания дашбордов и настройки алертов
- **Система безопасности**: Реализация защиты API от атак, аудита действий пользователей и логирования критических операций
- **Управление сессиями**: Система кэширования пользовательских сессий и управления состоянием приложения

#### 2.3 Интеграционные решения:
- **API для фронтенд-приложений**: Разработка RESTful API для интеграции с пользовательскими интерфейсами
- **WebSocket сервис**: Реализация real-time уведомлений и обновлений состояния системы
- **Интеграция с внешними системами**: API для взаимодействия с Docker, образовательными системами и другими внешними сервисами

### 3. Научные результаты:

- **Публикации в научных изданиях**: Статьи по результатам исследования микросервисных архитектур для образовательных платформ

- **Методические рекомендации**: Руководства по развертыванию, настройке и использованию разработанной платформы

- **Анализ эффективности**: Исследование производительности, масштабируемости и безопасности предложенных решений

- **Сравнительный анализ**: Оценка преимуществ микросервисной архитектуры по сравнению с традиционными монолитными решениями

### 4. Образовательные результаты:

- **Готовые лабораторные работы**: Набор практических заданий по информационной безопасности, адаптированных для использования на разработанной платформе

- **Обучающие материалы**: Документация, видеоуроки и методические пособия для преподавателей и студентов

- **Программы повышения квалификации**: Курсы для преподавателей по использованию современных технологий в образовательном процессе
